我：我现在学习UnityShader、3D图形渲染相关知识。手头上的文件夹包含《UnityShader入门精要》整书PDF，内含的官方源代码。我该怎样利用AI工具去辅助我学习？

Grok：你手头有《**Unity Shader 入门精要**》整书 PDF + 官方源代码文件夹，这是一个非常好的学习起点！Cursor（加上你已有的 AI Agent 功能）非常适合辅助学习 **Unity Shader** 和 **3D 图形渲染**，因为它能处理代码、解释原理、生成可视化、调试效果，还能把“书本知识”转化为**交互式学习**。

下面是**针对性、逐步可执行**的学习辅助方案，从基础到进阶，全部围绕 Cursor + 你的 PDF/源码。

#### 1. 先把 PDF 和源码准备好，让 AI “吃进去”
Cursor 目前**不能直接读取 PDF**，需要先转换：

- **推荐方法**（最简单有效）：
  1. 用免费工具把 PDF 转成 **Markdown**（推荐 **Marker** 或 **PDF to Markdown** 在线工具 / 本地运行）。
     - 重点：保留章节标题、代码块、数学公式（HLSL 代码要保持原样）。
  2. 把转好的 Markdown 文件（或分章节拆成多个 .md）放到你的源码文件夹里（和官方示例代码同目录）。
  3. 在 Cursor 中打开这个文件夹 → AI 会自动索引整个项目，包括所有 .shader、.cs、.md 文件。
  4. 聊天时用 **@文件名** 或 **@文件夹** 引用具体章节或代码。

- 额外技巧：
  - 把书的关键章节（如第 2 章渲染流水线、第 3 章 Shader 语法、第 6 章光照模型）单独拆成小文件，便于精准引用。
  - 如果想更高级，可以把 Markdown 上传到 GitHub Gist，然后在 Cursor 用 **@Docs** 添加（部分用户这样实现持久知识库）。

这样 AI 就能“记住”整本书的内容，结合官方源码一起理解。

#### 2. 设置专属 Rules（让 AI 变成你的 Shader 导师）
在 Cursor 设置里创建 **User Rule**（全局）或项目根目录的 `.cursor/rules/` 文件夹里新建规则文件。

**推荐 Shader 学习 Rules 示例**（直接复制进去）：

```
你是一位资深 Unity Shader 和 3D 图形渲染专家，精通《Unity Shader 入门精要》整书内容。
- 所有解释必须结合书中的章节和页码（例如“参考书第 6 章光照模型”）。
- 优先使用 Built-in Pipeline / URP / HDRP 的正确写法，并说明区别。
- 解释时必须包含：原理（渲染流水线中的位置）+ HLSL 代码片段 + 数学公式（用 KaTeX）+ 实际视觉效果描述。
- 生成 Shader 时，先给出完整 .shader 文件，再给出对应的 Material 设置和测试场景建议。
- 永远鼓励动手实践：每次回答后都提出 1-2 个小实验任务。
- 避免直接给完整答案，先引导思考或分步实现。
- 数学部分用清晰的向量、矩阵、点乘、叉乘解释。
```

保存后，所有 Ask / Plan / Agent / Debug 模式都会自动遵守，你的学习体验会非常连贯。

#### 3. 日常学习 Workflow（推荐每天这样用）
1. **阅读 + 理解（Ask / Plan 模式）**  
   - 选中书里一段文字（或 @章节.md），输入：  
     “用简单语言 + 图示解释这段内容在渲染流水线中的作用，参考书第 X 章，并指出常见错误。”
   - 或用 **Plan** 模式：让 AI 为你制定“本周 Shader 学习计划”（基于书目录 + 你的进度）。

2. **代码实践（Agent / Composer 模式）**  
   - 打开官方源码里的一个示例 Shader。
   - Cmd/Ctrl + K 或直接在 Agent 里说：  
     “基于书第 7 章法线纹理，实现一个带 Rim Lighting 的卡通 Shader，保留原有结构，只修改必要部分。”
   - AI 会直接编辑文件，你再在 Unity 中预览效果。

3. **调试与可视化（Debug 模式）**  
   - Shader 效果不对时，选中代码切换到 **Debug**，描述问题（“为什么这个高光在 URP 下变黑了？”）。
   - AI 会帮你插入调试输出、生成多个假设、建议修改。

4. **生成练习与测验**  
   - “根据书第 8 章透明效果，给我出 3 道渐进式练习题（从简单到复杂），并提供参考答案框架。”
   - 或 “把我刚写的这个 Shader 代码审查一下，指出不符合书中所讲最佳实践的地方。”

5. **进阶：多文件重构与实验**  
   - 用 **Composer**（Cmd/Ctrl + I）同时修改 Shader + C# 脚本 + Material，实现一个完整效果（比如水面反射 + 折射）。

#### 4. 结合 Unity 编辑器的额外增强
- 在 Unity 中把 **External Script Editor** 设置为 Cursor（Preferences → External Tools）。
- 安装 **Cursor Unity Package**（如果有 Git URL 方式），让 Cursor 能更好识别 Unity 项目。
- 一些用户还用 **Unity Editor MCP**（开源工具），让 Cursor/Claude 能直接控制 Unity 编辑器（创建物体、截图、运行 Play Mode），非常适合 Shader 视觉调试。

#### 5. 推荐的长期习惯
- 每天学一小节 → 用 AI 解释 + 自己改代码 → 记录到笔记（让 AI 帮你生成 Anki 卡片或总结）。
- 建立个人“Shader 知识库”文件夹：把书摘要、你的实验 Shader、AI 生成的解释都放进去，长期 @ 引用。
- 当你学到高级部分（Compute Shader、Ray Marching 等），用 **Subagents** 创建一个“图形数学专家”子代理，专门处理矩阵、向量运算。
